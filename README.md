# ☁️ LootOps Cloud

> A robust, **open-source**, self-hosted file storage and management platform.  
> Upload files, manage folders, share links, monitor your server in real-time, run Python scripts — all from a beautiful dashboard.

## ✨ Features

| Category | Features |
|---|---|
| 🔐 **Auth & Security** | Firebase Auth, 2FA (TOTP + Email OTP), Allowlist, Rate Limiting, Helmet.js |

| 🐍 **Python Studio** | In-browser Python IDE with live execution |
| 🖥️ **Live Telemetry** | Real-time CPU & RAM charts via SSE |
| 🌐 **Public Portal** | Public Explore page for public files |

## 🚀 Quick Start

### Option A — Interactive Setup Wizard (Recommended)

The easiest way to get started. Clone the repo, run the wizard, and follow the prompts:

```bash
git clone https://github.com/Subhan-Haider/Local-Cloud.git
cd Local-Cloud
npm install
npm run setup
```

The wizard will:
- ✅ Ask for your domain / hostname / IP
- ✅ Set API ports
- ✅ Walk you through Firebase credentials step by step
- ✅ Optionally configure email notifications (SMTP)
- ✅ **Auto-generate** a strong random API key and JWT secret
- ✅ Write your `.env.local` file automatically
- ✅ Show you the exact commands to run next

After setup completes, start the app:

```bash
npm run dev          # development
# or
npm run build && npm run start   # production
```

---

### Option B — Docker (Zero Node.js Required)

If you prefer Docker, you only need Docker installed on your machine:

```bash
git clone https://github.com/Subhan-Haider/Local-Cloud.git
cd Local-Cloud

# Run the setup wizard first to generate .env.local
npm run setup        # (requires Node.js just for setup)

# Then start with Docker
docker compose up -d
```

Dashboard: `http://localhost:3000`  
API: `http://localhost:5000`

```bash
docker compose logs -f     # view logs
docker compose down        # stop
```

---

### Option C — Manual Setup

If you prefer to configure manually:

```bash
cp .env.example .env.local
# Edit .env.local with your values (see Configuration section below)
npm install
npm run dev
```

## ⚙️ Configuration Reference

All configuration lives in `.env.local`. Run `npm run setup` to generate it interactively, or copy `.env.example` and fill manually.

| Variable | Required | Description |
|---|---|---|
| `SERVER_BASE_URL` | ✅ | Public URL of the Express API (e.g. `https://api.yourdomain.com`) |
| `NEXT_PUBLIC_API_URL` | ✅ | Same as above — used by Next.js to reach the API |
| `PORT` | ✅ | Express API port (default `5000`) |
| `UPLOAD_PATH` | ✅ | Directory to store uploaded files |
| `API_KEY` | ✅ | Secret key for programmatic access. Auto-generated by setup wizard |
| `JWT_SECRET` | ✅ | Secret for signing JWT sessions. Auto-generated by setup wizard |
| `FIREBASE_PROJECT_ID` | ✅ | Your Firebase project ID |
| `FIREBASE_CLIENT_EMAIL` | ✅ | Firebase Admin SDK service account email |
| `FIREBASE_PRIVATE_KEY` | ✅ | Firebase Admin SDK private key |
| `NEXT_PUBLIC_FIREBASE_API_KEY` | ✅ | Firebase web app API key |
| `NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN` | ✅ | Firebase auth domain |
| `NEXT_PUBLIC_FIREBASE_PROJECT_ID` | ✅ | Firebase project ID (public) |
| `NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET` | ✅ | Firebase storage bucket |
| `NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID` | ✅ | Firebase sender ID |
| `NEXT_PUBLIC_FIREBASE_APP_ID` | ✅ | Firebase app ID |
| `SMTP_ENABLED` | ❌ | Set to `true` to enable email notifications |
| `SMTP_HOST` | ❌ | SMTP host (e.g. `smtp.gmail.com`) |
| `SMTP_PORT` | ❌ | SMTP port (usually `587`) |
| `SMTP_USER` | ❌ | SMTP username / email |
| `SMTP_PASS` | ❌ | SMTP password or App Password |
| `ADMIN_EMAIL` | ❌ | Email that receives admin notifications |

## 🔥 Setting Up Firebase

Firebase is used for authentication. It's free for personal use.

1. Go to [console.firebase.google.com](https://console.firebase.google.com/) → **Create a project**
2. Enable **Email/Password Authentication** (Authentication → Sign-in method)
3. Enable **Firestore Database** (Build → Firestore Database → Create)
4. **Get Admin SDK credentials:**
   - Project Settings → Service accounts → **Generate new private key**
   - Open the downloaded JSON and copy the values into `.env.local`
5. **Get Client SDK credentials:**
   - Project Settings → General → Your apps → **Add a Web app**
   - Copy the `firebaseConfig` values into `.env.local`

> The `npm run setup` wizard walks you through this step by step.

## 🌐 Custom Domain / Hostname Setup

### Local Network Access (LAN)

To access from other devices on your Wi-Fi, use your machine's local IP:

```env
SERVER_BASE_URL=http://192.168.1.100:5000
NEXT_PUBLIC_API_URL=http://192.168.1.100:5000
```

Run `ipconfig` (Windows) or `ip addr` (Linux/Mac) to find your local IP.

### Custom Domain on a VPS

For a real domain (e.g. `cloud.yourdomain.com`):

```env
SERVER_BASE_URL=https://cloud.yourdomain.com
NEXT_PUBLIC_API_URL=https://cloud.yourdomain.com
```

Then set up Nginx as a reverse proxy:

```nginx
server {
    listen 80;
    server_name cloud.yourdomain.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl;
    server_name cloud.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/cloud.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/cloud.yourdomain.com/privkey.pem;

    # Dashboard (Next.js on port 3000)
    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

Get a free SSL certificate:

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d cloud.yourdomain.com
```

### Separate API Subdomain (Optional)

If you want the API on its own subdomain (e.g. `api.yourdomain.com`):

```env
SERVER_BASE_URL=https://api.yourdomain.com
NEXT_PUBLIC_API_URL=https://api.yourdomain.com
```

Add a second Nginx server block for `api.yourdomain.com` pointing to port `5000`.

## 🔑 First Login

On first run, the **Allowed Emails** list is empty — this means **any Firebase-authenticated user can log in**. After your first login:

1. Go to **Settings → Security → Allowed Emails**
2. Add your email address
3. Now only your email can log in — all others are blocked

## 🔒 Security Model

1. **Firebase Auth** — All sessions backed by Google Firebase
2. **Allowlist** — Optional email allowlist (empty = open mode, any Firebase user can log in)
3. **2FA** — TOTP (Authenticator app) or Email OTP
4. **Rate Limiting** — 100 requests / 15 min per IP
5. **Helmet.js** — Secure HTTP headers
6. **Path Traversal Guard** — File paths validated against uploads root
